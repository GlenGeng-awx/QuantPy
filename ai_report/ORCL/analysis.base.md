## 五、安全边际 — 静态锚不适用（g 质量护栏直接否决：烧钱式增长）

按 CLAUDE.md g 质量护栏：**`FCF − SBC = −23.69B − 4.81B < 0` → 烧钱式增长，不配用成长估值（min(8.5+g,30)）。** 与 SNAP 的 FCF−SBC<0 同理被排除——只是 ORCL 烧的是 capex（建产能）而非 SBC。

```
min(8.5+g,30) 在此失效：公式假设"E ≈ 股东价值"，但 ORCL 的 GAAP 盈利
背后 FCF −$24B、净债 +~$49B/年——盈利没转成股东现金，反而靠举债补。
用公式会严重高估（忽略 capex/债务黑洞）。
```

参照（仅说明"为何不能简单用 P/E"）：
- 若硬套 min(8.5+g,30)、g 封顶 30x → 合理价 ~$175（正常化 EPS $5.84）——**这个数会骗人**，因为它假装 FCF 不存在
- 真相：FCF 负 + 净债 $98-124B，资产/现金口径都无安全边际可言
- 唯一估值锚是"赌 RPO 转化 + OCI 利润率兑现"——那是成长投机，非价值
- **什么价位本框架才看**：需 **FCF 转正拐点**（证明 capex 赚回现金、RPO 转化为自由现金流）——届时才谈得上"好公司"，再按正常化盈利定价。在 FCF 转正前，任何价格都是杠杆增长投机

**关键风险**：OpenAI/Stargate 付款能力（RPO 含金量）、利率（巨额债的利息）、capex ROI。
