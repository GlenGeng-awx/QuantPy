# 选股分析框架

## 投资哲学

**格雷厄姆（好价格）**：安全边际足够大，即使判断错了也亏不多。

**费雪（好公司）**：GM 高且稳、护城河宽、FCF 强、管理层称职。不是什么便宜货都买，烂公司再便宜也不碰。

**巴菲特（一次性麻烦）**：把格雷厄姆和费雪连起来的"时机"。好公司平时不会给你好价格，只有在遇到暂时困难时才会。关键词是**暂时** — 困难会过去，公司还是那个公司。

三者缺一不可：
- 只有好价格没有好公司 = 价值陷阱（便宜的烂公司）
- 只有好公司没有好价格 = 买贵了（腾讯 P/E 16.5x）
- 好公司+好价格但麻烦不是一次性的 = 结构性恶化（BABA FCF 转负）
- 好公司+好价格+一次性麻烦 = **最佳买点**（ADBE P/E 11x + AI 恐慌 + CEO 过渡）

## 概述

本项目实现上述投资哲学的量化筛选框架。核心任务：识别"因暂时困境被市场低估的优质大公司"。

## 数据获取

数据由本仓库 fundamental 模块生成，Claude 可直接运行命令获取，无需用户粘贴：

```bash
# CHEAP + HEALTH SCORECARD + 原始财报（合并输出）
python3 -m fundamental.combine ADBE
```

当用户说"分析 ADBE"时，先运行上述命令获取数据，再执行四步框架。用户也可能直接粘贴输出，效果相同。

## 四步分析框架

### 第一步：价格筛选
满足任意1条即进入候选池：
- 1年回撤 >40%、2年回撤 >60%、距52周低点 <=15%
- P/E TTM <15x、EV/EBITDA <10x、P/S <2.0x（净利率>25%时放宽至4.0x）
- 金融股：P/B <1.0 替代 EV/EBITDA
- 周期股：用正常化P/E <12x、P/B <1.5x、净现金/市值 >30%、或股息率 >5%

交叉验证：EV/EBITDA <2x 但 P/E >10x 时，标记单位混淆风险。EV/EBITDA 至少查两个独立数据源。

### 第二步：估值分位验证
必须实时查询5年P/E（或P/S）分位数，当前须 <=30%。查询失败则暂停，严禁估算。周期股同时查 P/B 分位；P/E >70% + P/B <30% = 周期底部信号。

### 第三步：财务健康（SCORECARD + 财报 heuristic）

**SCORECARD 解读**：
权重：资产负债表 40%、现金流 35%、利润表 25%。
- 寻找"伤口模式"：3yr 高分 + TTM/5Q 低分 = 困境反转候选
- 区分"情绪伤口"（基本面完好，市场恐慌）与"结构性伤口"（真实竞争侵蚀）

**财报 heuristic 分析**（基于 STATEMENTS 原始数据，独立于 SCORECARD）：

利润表逐季拆解：
- 列出近 5 季的 Revenue / GM% / OpM% / Tax% / NM%，识别趋势和异常季度
- GM 连续下滑 = 定价权/成本结构恶化信号（如 PYPL 5季 -210bps）
- SGA 或 R&D 突然跳升 = 可能是收购费用（如 ADBE）或战略投入（如 BABA SGA +47.7%）
- 税率异常（偏高或偏低）会扭曲 EPS，需标注并正常化

GAAP 异常检查：
- 对比 Op Income 与 NI 走势：背离说明非经营项主导利润（如 BABA OpInc 转负但 NI 正）
- 识别一次性项目（重组、减值、诉讼和解、投资处置）并计算正常化 EPS
- Other Income Expense 大额波动 = 投资损益或 FX，不反映经营实力

资产负债表变化：
- Goodwill 跳升 = 收购（如 ADBE Q2 GW +$1.17B）
- 现金消耗速度：对比本年 vs 去年同期
- 新增债务 vs 回购力度的取舍变化

现金流质量：
- OCF/NI < 1 = 利润含金量低，警惕
- SBC vs NI：SBC 超过 NI 时（如 TTD），"真实利润"远低于报告值
- CapEx 趋势：急剧增加可能吞噬 FCF（如 BABA CapEx 3年翻4倍→FCF 转负）
- 回购金额变化：骤降说明管理层优先级转移（如 BABA 回购 -91%）

股份变化：
- 追踪 diluted shares 逐季变化，判断回购是否真正在缩股还是仅对冲 SBC

### 第四步：护城河、管理层与消息面

必须实时联网搜索（标注 URL 和日期）。按从稳定到变化的顺序：

**护城河**（最不易变，是长期持有的根基）：
- 壁垒类型：网络效应、切换成本、规模经济、品牌、专利/牌照
- 竞争格局：主要对手是谁、份额趋势是扩大还是收缩
- 威胁：新进入者、技术替代、监管变化

**管理层**（记录 fact 到 analysis 文件，便于跨期追踪）：
- 谁：CEO/CFO 姓名、任期、创始人还是职业经理人
- 利益绑定：内部人持股比例、近期公开市场增持/减持（金额+价格）
- 资本配置：回购时机是否聪明（低买 vs 高买）、M&A 历史成败、SBC 趋势
- 说到做到：过去 4 季指引 vs 实际的偏差
- 治理风险：管理层空缺、近期换帅、VIE 结构下的信息透明度

**消息面**（最易变，捕捉近期催化/风险）：
- 近3个月核心新闻（诉讼、回购、评级调整）
- 行业/宏观变化对标的的影响

**最后输出**：分别写"熊市逻辑"和"牛市逻辑"

## 输出格式

使用结构化模板和表格：
- 第一步：条件表格（通过/未通过）
- 第二步：当前值、5年区间、分位、数据来源
- 第三步：九宫格解读 + GAAP 异常项 + 正常化 EPS（如有 STATEMENTS）
- 第四步：熊/牛逻辑 + 来源 URL
- 综合判定：是否纳入候选池 + 仓位建议 + 建仓/加仓/止损价位

## 分析报告存档

每次完成分析后，将报告保存至 `financial_data/{STOCK}/analysis.{YYYY-MM-DD}.md`。

下次分析同一标的时，先读取最近一份报告，独立完成新分析后，对比前次报告标注关键变化（估值变动、评级调整、新风险/催化剂）。

多只股票对比分析时，将汇总报告（含对比表格和仓位建议）保存至 `financial_data/comparison.{YYYY-MM-DD}.md`。

## 关键分析偏好

- 货币：财报非美元时必须验证单位，报告开头展示换算表
- EV/EBITDA 交叉验证：手动核算 EV = 市值 + 债务 - 现金，与资产负债表数据核对
- 股东回报很重要：零回购+零分红是重大负面信号（尤其 VIE 结构）
- CEO 公开市场增持是最强看多信号；内部人卖出是弱看空（可能是税务/遗产规划）
- 多只股票对比时，使用统一维度的对比表格
- 仓位模型：10 只 × 10% 等权。核心决策是"是否值得占一个席位"，不搞小仓投机。分批建仓（先 5% 再 5%），但最终目标仓位就是 10%，不会更少也不会更多

## 代码风格（本仓库 Python 文件）

- 不用列表推导式 — 用显式 for 循环 + append
- 紧凑作用域内用短变量名（sr, su, sf 代表 stock_realized 等）
- 不写过长的单行表达式 — 拆成多行
- 极少注释，仅在 WHY 不明显时写
